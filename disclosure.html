Disclosure

FrameForge's developer contacted Digital Extremes directly to ask whether its features could place users at risk of account action. This page documents what was asked, what DE said, and what it means for you — transparently and in full.

⬇ Download Full DE Ticket (PDF)

Overview

FrameForge is a free, open-source companion application for Warframe. It reads your own inventory data and displays it to you — it never modifies the game, never automates gameplay, and never gives any competitive advantage.

Because some of its features interact with the Warframe process or DE's servers in unconventional ways, the developer (Sikewyrm) opened a formal support ticket with Digital Extremes on June 1, 2026 to ask whether any feature could put users at risk. The full exchange ran through July 16, 2026 and is documented on this page.

What FrameForge Does

FrameForge uses three methods to gather data. Here is exactly what each one does and its current status.

Feature 1

Memory Reading (ReadProcessMemory)

FrameForge uses the standard Windows ReadProcessMemory API to read your own inventory data from the Warframe process running on your machine. This is strictly read-only — the app never writes to game memory, never injects code, never hooks any function, and never modifies game state in any way. It only observes data that the game has already written into your machine's RAM to display your own items back to you.

This feature is disabled by default and must be manually enabled in Settings.
Feature 2

Public World State Data

FrameForge fetches live game data from DE's own public CDN endpoint (api.warframe.com/cdn/worldState.php) to power the Timers tab — void fissures, Baro Ki'Teer, world cycles, events, and more. No login or authentication is required. This is the same publicly accessible endpoint used by dozens of established community tools.

This feature is always active and requires no opt-in.
Feature 3

Warframe Inventory API

Suspended

This feature would optionally connect to Warframe's internal inventory API (api.warframe.com/api/inventory.php) using a temporary session token read from the Warframe process. The endpoint is not publicly documented by DE.

This feature is currently suspended. After the full exchange with DE's support team, we were unable to obtain a clear answer about whether accessing this specific endpoint is permitted. Rather than risk our users, we voluntarily disabled this feature pending clearer guidance. See Why Feature 3 is Suspended below.

The EULA

In their initial reply, DE's support team cited the following passage:

"You agree that you will not, under any circumstances: [...] reverse engineer, decompile, disassemble or otherwise derive source code or game data from the Software or reduce the Software to a human-readable form, except to the extent that such actions are expressly permitted by applicable law or publish or share same with anyone."

They attributed this to Section 2, subsection b. However, when we reviewed the current EULA at warframe.com/en/eula, that section structure no longer exists — the document appears to have been reorganised. The equivalent restrictions are now in Section 7 (Player Conduct).

We reviewed each relevant clause in Section 7 against what FrameForge actually does:

  • "intercept, emulate, or redirect any communication between the Services and us" — FrameForge does not intercept, emulate, or redirect any network communication between the game client and DE's servers.
  • "collect information about the Game" — FrameForge reads a player's own runtime inventory data from their own machine's local memory, not from DE's servers.
  • "mods, hacks, cheats, bots [...] that interact with the Services in any way" — FrameForge does not automate gameplay and does not interact with the game client in any way. It is a passive, read-only observer.
  • "datamine[ing]" — FrameForge does not extract data from game asset files or packages. It reads data the game has already loaded into RAM in a structured, readable format.

We asked DE to clarify which specific current provision applies. They were unable to provide a specific answer (see the exchange below).

Unanswered Questions

The following six questions were raised on June 6, 2026. DE's support team was unable to answer them and could not escalate to a team that could. They remain unanswered.

  1. EULA section: The section cited in DE's first reply (Section 2, subsection b) no longer appears in the current EULA. Which specific provision of the current EULA applies to FrameForge's described behaviour?
  2. "Collect information about the Game" (Section 7): Does reading a player's own runtime inventory data from their own machine's local memory — not from DE's servers, not via network interception — constitute "collecting information about the Game"?
  3. "Datamine" (Section 7): FrameForge does not extract data from game asset files or packages. Does reading data the game has already loaded into RAM fall within the definition of "datamining" as used in Section 7?
  4. "Non-public areas of the Services" (Section 7) — Feature 3 only: Does accessing the inventory API endpoint using the player's own active session token count as accessing "non-public areas of the Services"?
  5. "Leveraging only the functionality of the Game and Services" (Section 6): Does Section 6 prohibit companion tools that run entirely independently of the game client — tools that do not modify, hook into, or extend the game's functionality, but simply observe data on the same machine?
  6. Cheat Detection Software (Section 8): Should users be concerned that Warframe's cheat detection systems may flag FrameForge as an unauthorised process, given that FrameForge does not cheat, does not automate gameplay, and does not provide any competitive advantage?
These questions are documented here so users have the full picture. Where DE could not clarify, we have noted that outcome rather than speculating.

What This Means

DE's position is their standard response for all third-party companion tools — the same language used when players ask about tools like Warframe Companion, Overwolf overlays, and similar apps. It is important to understand exactly what was and was not said.

What DE confirmed

  • Players can run and develop third-party tools for Warframe.
  • Doing so is at the individual player's own risk.
  • DE does not support or endorse third-party tools.

What DE did not say

  • They did not say any FrameForge feature would result in a ban.
  • They did not instruct FrameForge to shut down or remove any feature.
  • They did not single out ReadProcessMemory or the inventory API as enforcement targets.
  • They did not tell us to stop distributing FrameForge.

In plain terms: DE's position is "using FrameForge is at your own risk." They have not said it is prohibited, and they have not ordered any action to be taken. DE's enforcement history targets players who cheat — aimbots, automation, gaining unfair in-game advantages over others. FrameForge does none of these things.

Why Feature 3 is Suspended

Feature 3 (the optional Warframe inventory API connection) was voluntarily suspended by us — not because DE told us to remove it, but because we were unable to get a clear answer about whether accessing an undocumented DE endpoint is permitted.

The six questions in the section above remain unanswered. Until we have a clearer picture — whether from DE directly, from the Warframe Forums, or from community precedent — we would rather err on the side of caution for our users.

If DE clarifies that this endpoint is acceptable for community tools, or if established community precedent shows it is used safely by other tools, we will re-enable the feature. If DE indicates it is not permitted, we will remove it entirely and say so clearly here.

Feature 3 is fully disabled in the app. Even if a previous version of FrameForge had it enabled in your settings, it will not make any API calls.

Summary

Feature Status Notes
Memory reading (Feature 1) Active Read-only. Same approach used by Overwolf, which has a formal partnership with DE. Disabled by default.
Public worldState data (Feature 2) Active Publicly accessible endpoint used by dozens of established community tools. Always active.
Inventory API (Feature 3) Suspended Voluntarily suspended pending clearer guidance from DE. No API calls are made regardless of settings.

Our Commitment

We will keep this page updated as the situation evolves. If DE's position changes — in either direction — we will update both this page and the app immediately and notify users.
