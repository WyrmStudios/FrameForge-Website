Documentation

Learn how to install and use FrameForge to track your Warframe inventory in real time.

Installation

System requirements

  • Windows 10 or Windows 11 (64-bit)
  • Warframe — Steam or standalone client
  • Typically requires no administrator privileges at runtime, though some system configurations may prompt for elevated access when reading game memory.

Steps

  1. Go to the latest release on GitHub.
  2. Download FrameForge_x64-setup.exe.
  3. Run the installer and follow the prompts.
  4. Launch FrameForge from the Start menu or desktop shortcut.
SmartScreen warning: Windows may warn that the app is from an unknown publisher. This is because code-signing certificates cost ~$300/year and FrameForge is a free, open-source project. Click More info → Run anyway to proceed. The full source code is on GitHub if you want to verify what you're running.

First Launch

You can launch FrameForge before or after starting Warframe — the order doesn't matter. The app detects the Warframe process automatically on startup.

Triggering an inventory refresh

Return to your Orbiter after completing a mission or visiting a relay. Your inventory refreshes within about 10 seconds.

Heads up: Opening the in-game ESC menu does not reliably trigger a scan. Returning to your Orbiter is the most consistent trigger.

Warframe API connection

Basic mod and arcane counts come from memory scanning and work without any account connection. Connecting the Warframe API adds mod ranks and additional detail. Credentials are scanned automatically from game memory — no manual login needed.

Inventory

The Inventory tab is the main view — everything you own, live. Items are organised into 13 categories:

  • All Owned, Resources, Mods, Relics, Arcanes
  • Warframes, Primary, Secondary, Melee, Companions, Archwing
  • Parts, Blueprints, Miscellaneous

Filters and search

  • Text search — filter by item name
  • Quick filters — Owned, Changed recently, Prime, Vaulted, Unvaulted
  • Sort — by quantity (high/low), name (A–Z / Z–A), or recently changed
  • Mod rank filters — filter mods by rank (Unranked through R30) when API data is available

Item card indicators

  • ★ — item is mastered (rank 30)
  • R{N} — partially ranked mod or arcane
  • ⚒ — currently building in Foundry
  • Green border — recently gained
  • Red border — recently consumed or spent

Change log

The bottom of the Inventory tab shows your full quantity change history, each entry showing the item name, category, delta (±), old and new quantity, and timestamp.

Data sources

Inventory quantities come from two sources that are merged together:

  • Memory scanner — reads stackable items (resources, blueprints, relics) and unique items (warframes, weapons, companions) directly from the game process every 10 seconds. No credentials needed.
  • Warframe API — when connected, adds mod ranks and additional detail on top of memory-scanned quantities.

Foundry

The Foundry tab lets you browse every craftable item in the game and track what you need to build it.

Component status

  • Gold (✓) — you own the built component or already have it.
  • Blue (BP) — you have the blueprint but not the built component.
  • Dark — you're missing both.

Relic source icons

If a missing component drops from a relic you currently own, a relic icon appears next to it so you know exactly which relic to run.

Additional indicators

  • Vault badge — 🔒 Vaulted / 🔓 Available based on the current drop table
  • ⚒ Crafting indicator — shown if a component is currently building

To track a build, star (★) it in the Foundry — it will appear in the Modular Window with a live shortage breakdown.

Modular Window

The Modular Window has four reorderable sections:

  • Tracking — crafting items starred in the Foundry, with per-item ingredient requirements and a Missing / All toggle.
  • Favorites — inventory items starred from the Inventory tab, shown with live quantities.
  • Timers — countdowns pinned from the Timers tab.
  • Watched Fissures — live void fissures matching your configured fissure watches.

All sections can be collapsed and reordered with up/down arrow buttons. The panel can be resized between 160 and 500 pixels wide, or detached as a floating popup window via Settings.

OCR Reward Overlay

When a void fissure reward screen opens in Warframe, FrameForge automatically captures and reads the reward cards and displays a transparent overlay — no alt-tab, no interruption.

What the overlay shows per card

  • Item name
  • Platinum sell price (from warframe.market)
  • Ducat value
  • 🔒 vaulted indicator if applicable
  • Set completion grid — which components of the set you own and which you're missing
  • A yellow arrow pointing to the recommended card based on your chosen priority mode

Pick priority modes

Set your preferred priority in Settings → Relic Overlay:

  • Item Completion (default) — picks the card most needed to complete a set you're building
  • Most Set Value (plat) — picks the card worth most when you complete the whole set
  • Most Plat (item) — picks the card with the highest individual platinum value
  • Most Ducats — picks the card with the highest ducat value

Display modes

The overlay uses PrintWindow (GDI) capture for Windowed and Borderless Windowed modes, and DXGI Desktop Duplication for Fullscreen Exclusive. Both are detected automatically.

Known edge cases: Rarity bar detection can occasionally miscount cards in unusual screen configurations or when other overlays interfere. The overlay also supports solo fissure missions (1 reward card). If you see incorrect results, check the session log in Settings → Support.

Market Helper

The Market tab shows every Prime set with live data from warframe.market. For each set you can see:

  • Total set platinum value
  • Per-part platinum price and ducat value
  • How many of each part you own (including duplicates)
  • Set completion percentage

Filters and sort

  • Filters: Has parts, Has duplicates, Complete sets, Fully owned
  • Sort: Name, Ducats (owned parts), Ducats (all parts), Completion %
  • Search: by set name

Prices are cached for the current session. No account required — requests go directly to the warframe.market public API.

Trading

The Trading tab connects FrameForge to your warframe.market account so you can manage listings and receive trade whispers without leaving the app.

Account required: Trading features require a warframe.market account. All other FrameForge features — Inventory, Foundry, Market prices, Relics, and Timers — work without any account.

Logging in

Enter your warframe.market email and password directly in the Trading tab — no browser redirect needed. Enable Stay logged in to save your session token to Windows Credential Manager. Your password is never stored.

Active Listings

View all your current sell and buy orders in one place. Edit the price or quantity of any listing, or delete it, without opening the warframe.market website.

Posting new orders

Click any prime set or individual part in the Market Helper tab to open a live order panel. From there you can see current sell and buy orders, a 3-week price history chart, and post a new listing with one click.

Trade Whispers

Incoming trade offers from warframe.market appear in the Messages tab, captured from your game's chat log via EE.log. One click copies a party invite to your clipboard, and you can mark the trade as sold directly from the message.

WFM Status

Set your warframe.market status — Online, In Game, or Invisible — from the Trading tab header. This controls your visibility to buyers and sellers on the site.

Relic Helper

The Relics tab shows every relic you own with its full reward table. For each reward slot you can see:

  • Item name and image
  • Rarity (Common / Uncommon / Rare) colour-coded as Bronze / Silver / Gold
  • Whether you own the item (✓) or have completed the full set (✓✓)
  • Platinum price and ducat value

Per-refinement counts (Intact, Exceptional, Flawless, Radiant) are shown for each relic so you know how many of each tier you have to spend.

Colourblind mode

Enable Settings → Colourblind mode to add text labels alongside the rarity colours.

Drop rate data is sourced from the community-maintained WFCD dataset and cached for 24 hours.

Timers

The Timers tab fetches live data from the official Warframe worldstate and refreshes every 60 seconds.

World Cycles

Cetus (Day/Night), Orb Vallis (Warm/Cold), Cambion Drift, and Zariman — each showing the current state and countdown to the next change.

Bounties

Reset timers for each open world: Cetus, Orb Vallis, Cambion Drift, Zariman, and Hex.

Daily & Weekly

  • Daily Reset and Weekly Reset countdowns
  • Sortie — expandable with all three missions and their modifiers
  • Archon Hunt — expandable with mission list
  • The Circuit — expandable with current Normal frame picks and Hard weapon picks
  • Deep Archimedea and Kahl / Break Narmer — weekly reset countdowns

Events

  • Baro Ki'Teer — arrival or departure countdown, with expandable inventory showing Ducat and Credit prices and Owned badges when he is present
  • Prime Resurgence — countdown with expandable inventory showing Aya prices and Owned badges
  • Nightwave, Darvo daily deal, and active community events

Alerts & Invasions

Each alert shows mission type, faction, reward, and countdown. Each invasion shows a progress bar, attacker and defender factions, and their respective rewards.

Void Fissures

Three tabs — Normal, Steel Path, and Void Storms — displayed as a grid of tiles with tier, mission type, faction, node, and countdown. Fissures matching a saved watch are highlighted in blue.

Fissure Watches

Configure a Mode + Tier + Mission Type filter. Matching fissures are highlighted in the fissure list and automatically shown in the Modular Window's Watched Fissures section.

Pinning timers

Star (☆) any timer row to pin it to the Timers section of the Modular Window for at-a-glance viewing.

Statistics

The Statistics tab has two sub-tabs: Trades and Reports.

Trades

Every completed in-game trade is detected automatically from Warframe's EE.log the moment the "Trade was successful!" message appears — no manual input needed. warframe.market trades are logged automatically when you click the Sold button in the Messages tab. A manual entry form is also available for any trade not captured automatically.

Each trade records:

  • Item name (including mod rank, e.g. "Serration (R10)")
  • Quantity and platinum per item
  • Trading partner's in-game name
  • Direction — sold or bought
  • Source — in-game, warframe.market, or manual
  • Timestamp

Trade history is stored locally in your FrameForge SQLite database at %LOCALAPPDATA%\warframe-companion\ and is never uploaded or shared.

Reports

The Reports tab turns your trade history into an analytics dashboard. Use the date range filter to scope everything to Today, 7 Days, 30 Days, 90 Days, Year to Date, 1 Year, or All Time. All charts and tables update instantly.

KPI cards

  • Total Earned — platinum received from sales
  • Total Spent — platinum paid for purchases
  • Net Profit — earned minus spent, with average per trade

Charts

  • Cumulative platinum line chart — your running net profit over time
  • Daily bar chart — green bars above the midline show days you earned, red bars below show days you spent; shows your trading rhythm at a glance

Tables

  • Items breakdown — every item you've traded, with sold count, bought count, average sale price, and net platinum; sorted by most profitable
  • Top trading partners — top 10 players by total platinum exchanged

Settings

  • Relic Overlay — enable/disable the OCR overlay and choose your pick priority mode
  • Colourblind mode — adds text labels to rarity colour indicators throughout the app
  • Text size — scale the UI between 80% and 140%
  • Modular Window — toggle pop-out mode to detach the side panel into a floating window
  • Account connection — manual Account ID + Nonce fields for Xbox/PlayStation players
  • Data — refresh item database, clear cached prices
  • View Log — view the overlay session log for debugging OCR issues
  • About — version number and update check (compares against GitHub releases)

FAQ

Is FrameForge safe to use?

FrameForge is strictly read-only. It uses ReadProcessMemory — the same Windows API used by Overwolf and hardware monitors. It never writes to memory, injects code, or hooks the game in any way.

Network requests go only to: warframe.market (prices, trading orders, and online status via WebSocket), the DE Warframe worldstate API (timers and events), WFCD community GitHub repositories (item catalog, recipe data, drop rates, localisation strings), and the Warframe CDN (item images). Warframe's EE.log is read locally to detect trade completions and fissure reward screens — it is never uploaded. No FrameForge server exists and no telemetry is collected.

Digital Extremes has historically permitted read-only companion tools. No account is required for inventory, Foundry, Market prices, Relics, Timers, or Statistics — trade detection and reporting work entirely from local data. A warframe.market account is only needed for trading features. The full source code is available on GitHub under GPLv3 — you can read every line and verify exactly what it does.

Does FrameForge track mods and arcanes?

Yes — mods and arcanes appear in the Inventory tab with per-rank copy counts when the Warframe API connection is active. Without the API, basic quantities are still available from the memory scanner.

Why doesn't my inventory update when I press ESC?

The ESC menu doesn't reliably put inventory data into memory in a scannable state. Return to your Orbiter after a mission or relay visit for the most consistent results.

The OCR overlay showed the wrong item — what happened?

The overlay uses Windows OCR and fuzzy text matching, which works well in most cases but can occasionally misread items — especially with diacritics or non-English text. If this happens, check the session log in Settings → View Log and open a GitHub issue with the log attached.

Will you add trading features?

Yes, trading features are planned for a future release.

Can I use FrameForge on Linux or Mac?

No. FrameForge uses Windows-specific APIs (ReadProcessMemory, WinRT OCR, DXGI) and is only supported on Windows 10 / 11.

How do I report a bug?

Open an issue on the GitHub Issues page. Include your Windows version, a description of what happened, and the overlay session log from Settings → Support if it's an OCR issue.

Known Limitations

  • Mission rewards (mid-mission) — in-mission reward data format is not identified; API data is only updated after a mission.
  • Rivens — not tracked.
  • Multi-monitor setups — DXGI capture targets the monitor containing the Warframe window, but unusual configurations may cause the OCR overlay to appear on the wrong display.
  • Non-English clients — the OCR fuzzy matcher works best with English item names; other languages may produce less accurate results.
  • Linux / Mac — not supported; depends on Windows-only APIs.
