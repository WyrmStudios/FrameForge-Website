Overview

FrameForge is a free, open-source companion tool for Warframe. After becoming more aware of the EULA and Terms of Service, its developer (Sikewyrm) contacted Digital Extremes directly to ask whether its features could place users at risk of account action. This page documents what was asked, what DE said, and what it means for you as a user.

What the EULA Says

Digital Extremes' End User License Agreement, Section 2 — License Restrictions and Limitations, states:

"You agree that you will not, under any circumstances: [...] reverse engineer, decompile, disassemble or otherwise derive source code or game data from the Software or reduce the Software to a human-readable form, except to the extent that such actions are expressly permitted by applicable law or publish or share same with anyone."

This is the clause most relevant to FrameForge. In plain terms: DE's EULA prohibits deriving data from the game software.

What FrameForge Actually Does

FrameForge uses three methods to gather data. Here is what each one does and how it relates to the EULA.

Feature 1

Memory Reading (ReadProcessMemory)

Low Risk

FrameForge uses a standard Windows API called ReadProcessMemory to read your own inventory data from the Warframe process running on your computer. This is strictly read-only — the app never writes to game memory, never injects code, never hooks any function, and never modifies game state in any way. It only observes data that is already in your machine's memory to display your own items back to you.

This feature is disabled by default and must be manually enabled in Settings.
Feature 2

Public World State Data

Effectively No Risk

FrameForge fetches publicly available game data from DE's own CDN endpoint (api.warframe.com/cdn/worldState.php) — the same one used by dozens of community tools like Warframe Status, Deimos Tracker, and others. No login or authentication is required. DE publishes it openly for exactly this kind of use.

This feature is always active and requires no opt-in.
Feature 3

Inventory API

Low Risk

FrameForge can optionally connect to Warframe's inventory API (api.warframe.com/api/inventory.php) using your own session credentials, which are temporarily read from the Warframe process memory while the game is running. Your password is never seen, stored, or transmitted by FrameForge — only a temporary session token is used, and it is held in memory only for the duration of the session. This is the same API endpoint used by DE's own mobile app.

This feature is disabled by default, requires manual opt-in, and displays an explicit risk warning before activation.

The Exchange with Digital Extremes

On June 1, 2026, Sikewyrm submitted a formal support ticket to Warframe Support (ticket #4235075) explaining all three features in detail and asking four direct questions:

  1. Does read-only use of ReadProcessMemory violate the EULA or Terms of Service?
  2. Is accessing the public worldState CDN permitted for community tools?
  3. Is accessing the inventory API with a player's own session credentials permitted?
  4. Will players who have already used these features face account action (warnings, suspensions, or bans)?
[DE]Tomasz June 4, 2026

DE stated that they do not support or endorse third-party software, and cited EULA Section 2b (quoted above). They did not directly answer whether users would face account action.

Sikewyrm (follow-up) June 4, 2026

Sikewyrm replied pressing specifically on the ban question, reiterating that FrameForge does not automate gameplay, does not provide competitive advantages, and does not interact with other players' accounts in any way.

[DE]Tomasz (final reply) June 5, 2026
"Please note, you can run/develop any third-party tool while playing Warframe at your own risk. There is no other info we can give you beyond our first response."

What This Means

DE's response is their standard position for all third-party companion tools — the same language used when players ask about tools like Warframe Companion, Overwolf overlays, and similar apps. It is important to understand what was and was not said.

What DE confirmed

  • You can run and develop third-party tools for Warframe.
  • The risk is yours as the individual player.

What DE did not say

  • They did not say any FrameForge feature would result in a ban.
  • They did not instruct FrameForge to shut down or remove any feature.
  • They did not single out ReadProcessMemory or the inventory API as enforcement targets.

Risk by Feature

Feature Risk Level Notes
Memory reading (Feature 1) Low Read-only. Same approach used by Overwolf, which has a formal partnership with DE.
Public worldState data (Feature 2) Effectively none Publicly accessible endpoint used by dozens of established community tools.
Inventory API (Feature 3) Low Opt-in only. No password handling. "At your own risk" per DE.
DE's enforcement history targets players who cheat — using aimbots, automating gameplay, gaining unfair advantages over other players. FrameForge does none of these things. It only reads your own data and displays it to you.

Our Commitment

If DE's position changes, or if they inform us that any feature poses a risk to users, we will disable that feature immediately and notify our users. The features most likely to be questioned (memory reading and the inventory API) are disabled by default specifically so that users make an active, informed choice before enabling them.

The full DE support ticket (ticket #4235075) is documented and preserved.
